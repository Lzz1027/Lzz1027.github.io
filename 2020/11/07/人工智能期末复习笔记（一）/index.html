Chapter3-Search

Key concepts in search 搜索的关键概念

  1. 我们能够达到的状态集合：包括初态(initial state)、目标状态(goal state)
  2. 对于每个状态，有我们可以进行的状态集；每个动作会导致一个新状态，由successor function定义，能够产生一个给定状态的能到达的所有状态
  3. Cost function决定了每个动作的花费
  4. Solution：从初态到目标状态的路径，Optimal solution，最小开销的解决方法

Generic search algorithm 通用搜索算法

Frontier：已经生成但未展开的节点集合（初态节点集合）

Loop：

  1. 如果Frontier是空的，声明搜索失败
  2. 从Frontier中选择并移除一个节点v
  3. 检查v的状态s是否是目标状态，如果是，声明搜索成功
  4. 如果不是，展开v，把v的展开节点放入frontier中

Key question in search：接下来要展开哪个生成的节点?

Uninformed search 盲目搜索

给定一个状态，我们仅能确定这个状态是否是目标状态；

不能确定一个非目标状态比另一个非目标状态看起来更好；

是否只能盲目地穿越状态空间，以期在某一点达到目标状态；

（盲目搜索并不意味着无组织的乱搜索）

Breadth-first search 广度优先搜索

节点的展开顺序与生成节点的顺序相同

​ Frontier可以保持为先进先出(FIFO)队列

BFS是完整的:如果存在解决方案，就会找到一个

BFS找到最浅的解决方案

​ 但不一定是最佳解决方案

如果每个节点有b个后续节点(分支因子)，第一个解的深度是d，那么frontier的大小在某一点至少是b的d次方

​ BFS算法要求更多的空间和时间

Deepth-first search 深度优先搜索

实现深度优先搜索

Frontier作为后进先出(LIFO)队列(栈)来维护

也容易实现递归

DFS(节点)

​ 如果目标(节点)返回解决方案(节点)

​ 对于节点的每个后续节点

​ 返回DFS(successor后续节点)，除非失败;

​ 返回失败;

深度优先搜索的属性

未完成(可能在非目标状态中循环)

如果找到解决方案，一般不是最优/最浅的

如果每个节点有b个后继节点(分支因子)，并且我们搜索最多深度m，边界最大值为bm

​ 有更好的空间需求

​ 实际上，通常甚至不需要存储所有的Frontier

时间:仍然需要查看每个节点

​ b的m次方 + b的(m-1)次方 +…+ 1 (for b> 1, O(b的m次方))

​ 对于不知情的搜索方法是不可避免的…

Limited depth DFS （有限深度搜索）

与DFS相似，只是当到达某个深度d时不再向更深的地方搜索

Iterative deepening DFS （迭代加深搜索）

​ 调用深度为0的有限深度DFS;

​ 如果不成功，则调用深度为1;

​ 如果不成功，调用深度为2;

​ ···

​ 完成，找到最浅的解决方案

​ DFS的空间要求

​ 可能因为重复的努力，DFS看起来很浪费时间，但真的没有那么浪费，因为几乎所有的努力都在最深层次

​ db + (d-1)b的2次方 + (d-2)b的3次方 +…+ 1b的d次方

​ 是O (b的d次) for b >1

Cost：搜索开销

BFS找到最浅的解决方案，因为总是先在最浅的节点上进行

​ 类似的想法：总是先在成本最低的节点上工作 (uniform-cost搜索)

​ 会找到最优解决方案(假设成本沿路径增加至少是恒定的)

​ 会先做很多小步

​ 如果最优成本为C，且每步成本至少增加L，则可以进入深度C/L

​ 与BFS类似的内存问题

从目标向后搜索

有时可以从目标回溯

​ 迷宫游戏

​ 8-puzzle

​ 到达位置F

​ “游遍所有地点”的目标是什么?

需要能够计算之前的节点而不是后继节点

Bidirectional search 双向搜索

同时从目标状态和初始状态平行搜索，如果最浅解决方法有深度d，并且分支因子在两个方向上都是是b，只要求O(b的d/2次方)个要被展开的节点

使双向搜索工作

需要知道边界是否相交

​ 需要在内存中至少保留一个frontier…

除此之外，可以对任意一棵树进行各种搜索，并得到相应的最优性等保证

如果向后搜索不可能(可行)则是不可能(可行)的

​ 很难计算前节点

​ 前驱分支因子高

​ 目标状态太多

Repeated states 重复状态

重复的状态会导致不完整或大量的运行时间

可以维护以前访问过的州的列表，以避免这种情况

​ 如果通往同一状态的新道路有更大的代价，就不要再追求了

​ 导致时间/空间的权衡

“忘记历史的算法注定会重蹈覆辙”——[Russell和Norvig]

Informed Search 启发式搜索

到目前为止，我们假设没有任何非目标状态比其他状态更好

但是在某些情况下，似乎有更好的非目标状态：

​ 即使不知道道路结构，有些地方似乎比其他地方更接近目标

​ 8-puzzle的一些状态似乎比其他更接近目标

首先扩展“看起来比较近”的节点是有意义的

启发式搜索过程

关键概念：heuristic function——启发式函数h(n)给出了从n到目标距离的估计

​ 目标节点的 h(n) = 0

例如：直线距离的旅行问题

假定：h (A) = 9, h (B) = 8 h (C) = 9, h (D) = 6, h (E) = 3, h (F) = 0

我们就给这个问题增加了一些新东西！这样就可以使用“启发式”来决定先扩展哪些节点

Greedy best-first search 贪婪最佳有限搜索

先扩展有最小h值的节点

能够迅速的找到最佳解决方法，但并不总是管用，如下例：

假定： h(A) = 9, h(B) = 5, h(D) = 6, h(E) = 3, h(F) = 0；

则会出现问题：贪心算法只计算节点剩余的估计值，而不考虑已经发生的代价

A*算法

让g(n)表示已经路径到达n时已经发生的代价

先扩展有最小g(n)+h(n)的节点

假定： h(A) = 9, h(B) = 5, h(D) = 6, h(E) = 3, h(F) = 0；

则在选择B或D时，g(B)+h(B)=7+5=12，g(D)+h(D)=3+6=9，就会选择更合适的那条路

如果h处处为0，那就是均匀代价搜索

可容许的：

启发式是可容许的，如果它从不高估目标的距离

​ 如果n是从n’出发可达的最优解，则g(n)≥g(n ‘) + h(n ‘)

直线距离是可以接受的:除了通往目标的一条直线之外，别无他求

可采纳的启发式意味着A*总是乐观的？？？？

一致性

如果以下条件成立，启发式是一致的:

​ 如果一步操作，将我们从n’带到n，那么h(n’) ≤ h(n) + 从n ‘到n的一步代价

​ 类似三角形不等式

​ 相似的，g(n ‘)+h(n ‘) ≤ g(n)+h(n)

意味着可容许的

迭代深化A*

A*算法的一大缺点是空间要求：类似于均匀代价搜索，BFS

Limited-cost depth-first A

代价有限深度优先A*

​ 给定某个代价截止c，任何具有g(n)+h(n) > c 的节点不展开，否则执行DFS
