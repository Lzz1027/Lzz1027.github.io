Chapter 8 First-Order Logic：

命题逻辑的局限性

到目前为止，我们学习了命题逻辑

英语中有些语句很难用命题逻辑建模:

“如果你的室友被雨淋湿了，你的室友一定一把伞也没带。”

可悲的尝试:

RoommateWetBecauseOfRain =>
(不是(RoommateCarryingUmbrella0)和
不是(RoommateCarryingUmbrella1)和
不是(RoommateCarryingUmbrella2)和…)

命题逻辑的问题

没有对象的概念

没有对象之间关系的概念

RoommateCarryingUmbrella0对我们很有指导意义，

建议
有一个物体我们称之为室友，

​ 有一个物体我们叫它Umbrella0，

​ 这两个物体之间有承载关系

正式地说，这里面没有以上的任意一种意思

​ 也许可以用 P 代替 RoommateCarryingUmbrella0

Elements of first-order logic 一阶逻辑的元素

Objects：能给像雨伞、人、John、地球之类的事物名字

Relations：Carrying（Person0，Umbrella0)，IsAnUmbrella(Umbrella0)

​ 和某个Object的关系 = unary relations (一元关系) = properties

Functions：Roommate（.)

​ Roommate(Person0)

Equality：Roommate(Person0) = Person1

关于函数需要注意的事情

对每个函数要有单独的名字如 Roommate(Person0)

例如，室友(Person0) = Person1

……但我们不需要有这样的名字

一个函数可以应用于任何对象

​ 如室友(Umbrella0)

同时对许多物体进行推理

变量:x, y, z，…可以引用多个对象

新操作符 “对于所有” 和 “存在”

​ 全称量词和存在量词

for all x: CompletelyWhite(x) => (PartiallyBlack (x))

​ 完全白色的物体永远不会部分黑色
there exists x: PartiallyWhite(x) AND PartiallyBlack (x)

​ 世界上存在着部分白色和部分黑色的物体

练习将英语转换为一阶逻辑

“约翰有简的雨伞”
Has(John, Umbrella(Jane))

“约翰有一把伞”
there exists y: (Has(John, y) AND IsUmbrella(y))

“任何有伞的东西都不会湿。”
for all x: ((there exists y: (Has(x, y) AND IsUmbrella(y))) => NOT(IsWet(x)))

“任何有伞的人都不会湿”
for all x: (IsPerson(x) => ((there exists y: (Has(x, y) AND IsUmbrella(y))) => NOT(IsWet(x))))

公理和定理

公理：关于领域的基本事实，我们的“初始”知识库

定理：逻辑上从公理推导出的陈述
