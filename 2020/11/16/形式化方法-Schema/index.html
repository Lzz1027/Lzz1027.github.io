Schema计算

模式可以看作是单元，由各种类似于逻辑操作符的操作符操作。

模式操作符：

  • Decoration 装饰

  • Inclusion 包含

  • Conjunction 结合

  • Disjunction 析取

  • Delta convention 德尔塔约定

  • Xi convention Xi约定

  • Renaming 重命名

  • Hiding 隐藏

  • Projection 投影

  • Composition 组合

Decoration 装饰

用’ 修饰模式名S，变为S’，定义为与所有变量都用 ‘ 修饰的模式名S相同。

它用于表示执行了某些操作后模式的值。

Inclusion 包含

schema的名称可以包含在另一个schema的声明中。

这样做的效果是在文本上导入被包含的schema:它的声明与包含模式的声明合并，它的谓词部分与包含模式的谓词部分合并。

Conjunction 结合

两个schema S 和 T 可以通过一个模式连接操作符 and 来连接，其编写方式类似于逻辑连接操作符。

其效果是创建一个新模式，其中合并了两个组件模式的声明，并将它们的谓词结合在一起。

Disjunction 析取

两个schema S 和 T 可以通过模式析取操作符 or 来连接，就像逻辑析取操作符一样编写。

其效果是创建一个新模式，合并两个组件模式的声明，分离它们的谓词。

Renaming 重命名

schema中的观察可以通过以下形式重命名:

newSchemaName == oldSchemaName[newName1/oldName1, newName2/oldName2, …]

Hiding 隐藏？？？？？？？？？

schema隐藏操作符隐藏指定的变量，这样它们就不再是模式的变量，而只是成为schema谓词部分的存在操作符的局部变量。

newSchemaName == oldSchemaName \ (varName1, varName2, …)

Bhidden == S \ (b)

Projection 投影？？？？？？？

schema投影类似于隐藏，除了已命名的变量之外，所有变量都被隐藏了。

newSchemaName == oldSchemaName ↑ (varName1, varName2, …)

Aprojected == S ↑ (a)

Composition 组合？？？？？？？

schema S和schema T的组合写成 S ; T 并且表示做S，然后做T的效果。

这相当于将描述S后状态的变量重命名为一些临时名称，并使用相同的临时名称将描述T前状态的等效变量重命名，然后隐藏临时名称。

Example：

PressRight == CursorControlKey [key?: KEY | key? = right]

PressLeft == CursorControlKey [key?: KEY | key? = left]

PressRight ; PressLeft == (PressRight [tempCol/column’, tempLine/line’ ] ∧ PressLeft [tempCol/colimn, tempLine/line]) \ (tempCol, tempLine)

输入输出变量

输入变量

变量名后面加上问号(?)表示该变量是schema的输入。

输出变量

变量名后面加上感叹号(!)表示该变量是schema的输出。

Example：

KEY ::= home | return | left | right | up | down

CursorControlKey == HomeKey ∨ ReturnKey ∨ LeftKey ∨ RightKey ∨ Upkey ∨ Downkey

课程作业:模块学生项目

本规范涉及模块课程的学生。学生从提供的模块中选择模块，并通过添加和删除模块来构建一个方案。如果该方案满足某些条件，它就是可行的。至少必须有一个可行的计划。
