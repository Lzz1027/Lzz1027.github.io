Z 中的集合

Basic Types

基本类型也称为给定类型。

规范的基本类型声明时并不关心它们的实际元素是如何表示的。

例如，规范可能引用所有可能的汽车注册集，而不考虑这些注册是如何表示为字符的。

基本类型的声明方法是将其名称写在方括号中，并用注释表示其意图。

这套车辆登记可称为登记，并以书面形式:

​ [登记] 所有可能的汽车登记的集合

按照惯例，基本类型的名称完全用大写字母书写，使用单数名词。

另一种基本类型可能是所有人的集合:

​ [PERSON] 所有人的集合

在一行中可以列出几种类型:

​ [登记、人]

一般来说，基本类型的选择应该尽可能广泛。

而且，应该假设类型的元素是惟一可标识的。

Z中的谓词和量词

谓词

​ 0-ary predicates 零元谓词可以被看作propositions命题(sentences句子)，因为它们是独立于任何单个变量的事实的简单陈述。

​ Unary predicates一元谓词只是对象(个体)的属性。

​ Binary predicates二元谓词是对象对(个体)之间的关系，一般n元谓词表示对象的n元组(个体)之间的关系。

量词

​ 全称量词，存在量词。

​ 唯一量词，计数量词。

全称量词

​ 全称量词写为，发音为“for all”(看起来像for all中的颠倒的“A”)。它的用法是:

​ for all 声明|约束•谓词

​ 表示对于声明中给出的变量，限制为特定值(约束，’ | 约束’可以省略)，谓词适用于所有值。

存在量词

​ 存在量词写为，发音为”it looks like a backwards“(它看起来像there Exists中的倒着的’ E ‘)。它的用法是:

​ E(反) 声明|约束•谓词

​ 声明对于声明中给出的变量，限制为特定值(约束，’ | constraint ‘可以省略)，谓词适用于某些值。

唯一量词

​ 唯一量词被写为，与存在量词相似，除了表示只有一个谓词为真值的值。

计数量词

​ 计数量词对谓词包含的变量的值进行计数。

​ 在Z中这是不需要的;我们使用集合理解来构造谓词所包含的值的集合，然后查找集合的大小。

集合理解

​ 它通过给出一个条件(谓词)来构造(定义)一个集合，该条件必须适用于集合的所有成员。

​ 集合理解的一般形式是

​ {声明|约束•表达式}

​ 其中，声明为典型元素，并给出元素的类型;

​ 约束(“|约束”可以省略)限制典型元素的可能值，它是一个逻辑表达式，该逻辑表达式必须为真，才能包含该典型元素的值;

​ 表达式是指示要包含在集合中的值的表达式。

例子：

​ {x: Z |Even(x) • x * x} 偶数的平方的集合

​ {x: Z • x * x} 整数的平方的集合

Z中的关系（Relation）

关系是一个集合

​ 到目前为止所考虑的示例都局限于一种基本类型，这意味着规范不能开发得更复杂。我们需要的是将集合与其他集合联系起来的方法。这可以通过一个关系来实现，这个关系是基于笛卡尔积的思想。

笛卡尔积（Cartesian product）

​ 笛卡尔积，以法国数学家笛卡尔的名字命名，是两个或多个集合的值对。集合或类型X, Yand Z的笛卡尔积为:

​ X × Y × Z

​ 读作“X、Y和Z的笛卡尔积”，或“X叉乘Y叉乘Z”，从这个组合中得到的值称为元组，写作：（x，y，z），其中，x为类型X，y为类型Y，z为类型Z。

​ 举例：

​ {red, green} x {hardtop, softtop) = {(red, hardtop)， (red, softop)， (green, hardtop)， (green, softtopl)}

​ 这样的元组称为有序元组，因为编写组件的顺序很重要。

​ 由两种类型组成的元组称为对，而由三类型组成的元组称为三元组。n种类型的元组有时称为n元组。

​ 元组(4元组)的一个连接有关一个人的信息的例子:

​ Name x ADDRESS x N x TELEPHONE

​ 来记录姓名、地址、年龄和电话号码。

定义域 Domain

​ 在大多数情况下，关系只涉及到源的一个子集。这个子集称为定义域domain，写作“dom”。

​ 在这个例子中R的定义域是

​ 它是包含X的值的子集，这些值是由R将X和Y联系起来的。

值域 Range

​ 通常一个关系中只涉及目标的一个子集。这个子集叫做值域range，写作ran。

​ 在这个例子中，关系R的值域是

​ ran R = {y2, y3}

​ 它是包含Y的值的子集，其中R至少与X的一个值相关

函数是一种关系

关于创建规范的提示

介绍

​ 创建正式规范的过程需要的不仅仅是熟练使用Z符号提供的数学工具。它需要想象力和调查能力，以及在出现困难和出现新方法时修改自己工作的能力。

​ 有人说，对正式规范最重要的帮助是一个大的废纸篓!

​ 本章提供了一些关于如何创建正式规范的提示。

类型

​ 找出系统中要指定的集(类型)是什么。保持这些集合一般化。使用“所有…的集合”允许您考虑一个子集，并从”所有的集合“添加新的元素到子集，并从子集删除元素。

​ 例如，如果你声明：

​ [STUDENT] the set of students at this university

​ 由于类型是固定的，你不能招收任何新生或允许学生离开。

​ 下面的就好得多了:

​ [PERSON] the set of all persons

​ students: PPERSON

​ 确保您的类型是真正原子的。

​ 例如，不要声明表示一组学生的类型类。

​ 相反，使用上面提到的PERSON类型并使类成为一个子集

关系

​ 接下来考虑类型之间的关系。通过发现什么类型的多少个值与其他类型的多少个值相关，可以发现是否有任何关系是函数。

​ 如果是，这些函数是全部的，部分的，还是单射的?

​ 你发现的关系和函数之间有什么关系吗?

​ 例如，它们必须具有相同的域，或者一个域必须包含另一个域的范围?

​ 如果某些值有自然顺序，那么可以考虑使用序列;

​ 否则，使用集合会更容易。记住，陈述显而易见的东西是必要的!

状态

​ 这种关系的研究将导致一个表示状态的图式。

​ 您还将发现一些不变量，它们包含在state模式中。

初始化操作

​ 您需要提供一个初始化操作来设置初始状态。

​ 初始状态应该尽可能简单。

​ 通常它会给出空集和关系作为起始点。

​ 重要的是，初始状态不应违反系统的任何不变量。

​ 理想情况下，这将得到正式证明。对于一个简单的初始状态，通常很容易看出不变量没有被违反。

操作

​ 首先只考虑每个操作在给定合理值时的行为。

​ 检查值是否合理(这是先决条件)，但在此阶段不处理错误。

​ 在后面的阶段，定义一个单独的模式，通过返回一个应答值来处理错误。

​ 不要忘记显式地声明操作的结果没有改变什么.

查询操作

​ 查询操作不会改变状态，因此不能违反不变式。尽可能使用xi (Ξ)模式。
