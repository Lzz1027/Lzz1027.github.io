Chapter 5 Games：

博弈

在AI中创建游戏程序有的丰富传统

博弈与搜索有很多相似之处

大多数游戏研究有以下特征：

​ 有两个玩家

​ 是零和游戏：一方赢，另一方输

​ 拥有完美的信息：游戏的整个状态对游戏双方在任何时刻都是可知晓的

例如：井字棋，跳棋，国际象棋，围棋，西洋双陆棋，…

我们现在主要关注以上类型的游戏博弈。

最近的研究对其他游戏更感兴趣，特别是没有完全信息的游戏，比如扑克

​ 需要概率论、博弈论来进行此类博弈

“Sum To 2” Game “加到2” 游戏

玩家1行动，然后是玩家2，最后是玩家1，总计三次行动

每次行动的Move值为 0或1

当且仅当所有走法的Move和为2时，玩家1获胜

可能的博弈结果如上图，叶子节点表示了玩家1的胜负情况，1表示胜利，-1表示失败

Backward induction 逆向归纳法(又名 minimax)

使对方得分减到最低以使自己得最高分的战略

从叶子节点向上，分析玩家在节点的最佳决策，给节点一个值

​ 一旦知道节点的价值，容易找到最佳行动(选择最佳价值)

修改游戏？

​ 从叶子向上，分析玩家在节点的最佳决策，给节点一个值

一个递归实现：

Value(state)

If state is terminal, return its value

If (player(state) = player 1)

​ v := -infinity

​ For each action

​ v := max(v, Value(successor(state, action)))

​ Return v

Else

​ v := infinity

​ For each action

​ v := min(v, Value(successor(state, action)))

​ Return v

不需要查看所有的叶子节点

还需要查看下面两图中的问号标记的节点的值吗？

Alpha-Beta Pruning α-β 剪枝

修剪 = 砍掉搜索树的一部分(因为你意识到你不需要查看它们)

​ 当我们考虑 A* 时，我们也修剪了搜索树的大部分

保持 alpha = 目前路径上玩家1的最佳选择值

​ Beta =目前为止玩家2的最佳选择值

Pruning on beta 在β上剪枝

β在节点v处的值是-1

我们知道节点v的值至少是4，所以-1路径将是首选

不需要进一步研究该节点

Pruning on Alpha 在α上剪枝

α在节点w处的值是6

我们知道节点w的值至少是-1，所以6路径将是首选

不需要进一步研究该节点

修改递归实现来进行α-β剪枝

Value(state, alpha, beta)

​ If state is terminal, return its value

​ If (player(state) = player 1)

​ v := -infinity

​ For each action

​ v := max(v, Value(successor(state, action), alpha, beta))

​ If v >= beta, return v

​ alpha := max(alpha, v)

​ Return v

​ Else

​ v := infinity

​ For each action

​ v := min(v, Value(successor(state, action), alpha, beta))

​ If v <= alpha, return v

​ beta := min(beta, v)

​ Return v

α-β剪枝的好处

没有修剪，需要检查O(b的m次方)个节点

有修剪，取决于我们首先考虑哪个节点

如果我们选择一个随机后继节点，则需要检查O(b的3m/4次方)个节点

如果我们首先选择最好的继任者，需要检查O(b的m/2次方)个节点

​ 选择下一个后继节点的实际启发式，非常接近这个

可以有效地观察到两倍深!

​ 合理游戏和专家游戏的区别

重复的状态

在搜索中，多个动作序列可能会导致相同的状态

同样，可以跟踪以前看到的状态(在这里通常称为换位表)

​ 可能不想跟踪所有以前看到的状态…

使用 evaluation functions 评价函数

大多数游戏都太大，即使通过α-β剪枝也无法解决

解决方案:只向前看有限的深度(非终端节点)

通过启发式评估深度截止节点(又名评价函数)

如象棋:

​ 物质价值:皇后值9点，车5点，主教3点，骑士3点，兵1点

​ Heuristic 启发式：玩家物质价值的差异

解决这一问题

尝试评估节点是否处于静止状态 quiescent

 quiescent = 评估函数在近期内不会发生快速变化

​ 仅对静止节点应用评估函数

如果在某个状态有“明显的”移动，在应用评估函数之前应用它

与不太优秀的球员比赛

极小极大对于其他极小极大玩家是最优的

如何对付那些以其他方式打球的球员?

多玩家，完全信息的一般和博弈

基本的逆向归纳仍然有效

​ 不再叫极大极小

“自然”的随机移动游戏

例如，骰子游戏(自然选择掷骰子)

逆向归纳仍然有效……

求值函数现在需要基数正确(而不仅仅是普通正确)

不完全信息博弈

玩家不一定能看到整个游戏当前状态

​ 纸牌游戏

极其简单的扑克游戏:

​ 玩家1摸到K(赢)或J(输)，

​ 玩家1可以raise加注或check过，

​ 玩家2可以call看牌或fold弃牌

虚线表示无法区分的状态

逆向归纳不工作，需要随机策略的最优!

​ (稍后课程将介绍)

随机策略需要的直觉

假设我的策略是“摸到 K 就加注，摸到 J 就看牌”

​ 你要做什么?

​ 你期望的效用是什么?

如果我的策略是“总是加注”呢?

如果我的策略是“摸到 K 就加注，10%的概率摸到 J 加注”会怎样?

一些游戏的技术水平

国际象棋:

​ 1997年:IBM“深蓝”击败卡斯帕罗夫

​ 关于电脑是否真的更好，仍然存在一些(非常有限的)争论。

​ 不过现在，如果人类在比赛中想出了意想不到的绝招，人们就会怀疑他们秘密使用了电脑!

跳棋:

​ 1994年以来的计算机就成为了世界冠军

​ 人们仍然在争论电脑是否真的更好(“如果廷斯利还活着，电脑就不会打败他!”）

​ “怎么反驳呢?”

​ 直到2007年：跳棋被电脑完美的解决了

围棋：

​ 分支系数真的很高，似乎会在一段时间内无法触及

​ 然后AlphaGo出现了，优于人类棋手

扑克:

​ AI现在在2人(“抬头”)游戏中击败人类顶级玩家

​ 3+玩家的情况更不容易理解

这对社会有什么价值吗?

一些为游戏开发的技术在其他领域也有应用

​ 特别是“对抗”设置

但现实世界的战略局势通常不是两方的，完全信息的，零和的，…

但博弈论不需要这些
