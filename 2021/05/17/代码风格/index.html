1. EditorConfig

EditorConfig 并不是软件或插件，仅仅是一个名为 .editorconfig 的自定义文件，该文件用于定义项目代码规范，编辑器行为会与其保持一致，且其优先级甚至高于编辑器自己的设置（在 VsCode 中需要安装插件支持）。

2. Husky

Husky 是 git 的代码规范工具，能够防止不规范代码被commit、push、merge等

安装husky：

Ant-Design-Vue 中的 .huskyrc

3. Eslint

Eslint 是一个语法规则和代码风格检测工具，能在项目编译时提醒语法或风格错误，用于保证书写语法正确，风格统一的代码。

配合 VsCode 插件，Eslint 也可以在书写代码时就进行提示，帮助修正语法风格。

但 Eslint 通常只提供检测和提示功能，并不能直接格式化代码，还需要 pritter 等格式化工具的辅助。

Ant-Design-Vue 中的 .eslintignore ，用于标注不需要使用 Eslint 的内容：

Ant-Design-Vue 中的 .eslintrc ，详细规定代码风格，可以引入其他规定，如本项目中，对 typescript 以及 typescriptX 文件即采用 @typescript-eslint 规范；

4. Stylelint

StyleLint 是一个强大的 CSS 代码风格检测工具，可以规范 CSS 书写样式及顺序，且支持 Less 、Sass 等 CSS 预处理语言，有大量第三方插件，在 Facebook、Github 等公司得到实践，能覆盖很多使用场景。

Ant-Design-Vue 中有两个配置文件，查询发现好像都可以生效，逐条看并没有冲突项，一般情况下应该是只需要一个配置项即可。

Ant-Design-Vue 中的 .stylelintrc

Ant-Design-Vue 中的 .stylelintrc.json

5. Pritter

Pritter 是一个代码格式化程序，能够解析代码并按规则格式化，可以手动格式化也可以经配置在保存时自动格式化，可以配合 Eslint、Stylelint 使用。

Ant-Design-Vue 中的 .prettierignore

总结

整个 Ant-Design-Vue 项目经过如上配置，包含个人本地的多种文件语法风格检测、格式化，以及 git 协作编程时的二次检测，增强了代码的可读性和可维护性，使得团队的代码风格保持一致。
